# <a name="service-request-sr-information"></a>Hizmet isteği (SR) bilgileri
Herhangi bir Mobile Engagement hizmeti için bir hizmet isteği açtığınızda, aşağıdaki bilgileri sağlayın:

## <a name="ids-applicable-identifiers-related-to-your-issue"></a>Kimlikleri: geçerli tanımlayıcıları, konuyla ilgili:
* Uygulama Kimliği
* Kampanya kimliği
* Cihaz Kimliği
* Kullanıcı Kimliği
* Kullanıcı adı
* Uygulama bilgileri etiketi

## <a name="errors-applicable-error-information"></a>Hata: Geçerli hata bilgileri
* Sorunun nerede oluştuğunu API veya UI bölümün adı
* Aldığınız herhangi bir hata iletisi metni
* Gelen gerçekleştirdikten herhangi bir test sonuçlarını [sorun giderme kılavuzları](http://go.microsoft.com/fwlink/?LinkId=524382)

## <a name="code-applicable-coding-information"></a>Kod: Uygun kodlama bilgileri
* SDK sürümü ve platform uygulamanız (Android SDK 2.4.1, iOS 1.16.2, vs.)
* Üretim uygulamanız (veya geliştirme uygulamanızı APK/TGZ dosyaları) konumunu indirin
* "AndroidManifest.xml" ve/veya hiçbir kod parçacığını uygulamanızdan Azure Mobile Engagement için ilgili (Gelişmiş sorun giderme)
