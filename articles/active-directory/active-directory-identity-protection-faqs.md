# <a name="azure-active-directory-identity-protection-faq"></a>Azure Active Directory kimlik koruması ile ilgili SSS

Bu makale, Azure Active Directory (Azure AD) kimlik koruması hakkında sık sorulan soruların yanıtlarını içerir. Daha fazla bilgi için bkz: [Azure Active Directory kimlik koruması](active-directory-identityprotection.md). 


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Neden bazı risk olayları "Kapalı (Sistem)" durum var mı?

**Y:** bu Azure Active Directory kimlik koruması algıladı ve olayları artık riskli olduğu kabul için daha sonra kapattı olaylardır. Bu olaylar, kullanıcının risk düzeyi sayılmaz. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>Azure portalında kimlik koruması kullanmak için genel yönetici olmanız gerekiyor mu?
**C:** Hayır. Bir güvenlik Okuyucu, bir güvenlik yöneticisi veya genel yönetici kimlik koruması kullanmayı ya da olabilir.

---

## <a name="how-do-i-get-identity-protection"></a>Kimlik koruması nasıl sağlarım?

**Y:** bkz [Azure Active Directory Premium ile çalışmaya başlama](active-directory-get-started-premium.md) bu yanıtını için.

---

## <a name="how-can-i-sort-users-in-users-flagged-for-risk"></a>"İçin risk bayrak eklenen kullanıcılar" kullanıcıların nasıl sıralama yapabilirsiniz?

**Y:** tıklayarak risk rapor için bayrak eklenen kullanıcılar karşıdan **karşıdan** tepesindeki **bayrak eklenen kullanıcılar için risk** sayfası. Daha sonra son güncelleştirilen (UTC) dahil olmak üzere kullanılabilir alanlara göre karşıdan yüklenen veri sıralayabilirsiniz.
