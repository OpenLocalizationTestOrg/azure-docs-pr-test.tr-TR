## <a name="example"></a><span data-ttu-id="5ff11-115">Örnek</span><span class="sxs-lookup"><span data-stu-id="5ff11-115">Example</span></span>
<span data-ttu-id="5ff11-116">Aşağıdaki kodu kullanarak sorgulama yapmayı çok parça kullanımını göstermektedir bir verilen **ShardMap** adlı *myShardMap*.</span><span class="sxs-lookup"><span data-stu-id="5ff11-116">The following code illustrates the usage of multi-shard querying using a given **ShardMap** named *myShardMap*.</span></span> 

    using (MultiShardConnection conn = new MultiShardConnection( 
                                        myShardMap.GetShards(), 
                                        myShardConnectionString) 
          ) 
    { 
    using (MultiShardCommand cmd = conn.CreateCommand())
           { 
            cmd.CommandText = "SELECT c1, c2, c3 FROM ShardedTable"; 
            cmd.CommandType = CommandType.Text; 
            cmd.ExecutionOptions = MultiShardExecutionOptions.IncludeShardNameColumn; 
            cmd.ExecutionPolicy = MultiShardExecutionPolicy.PartialResults; 

            using (MultiShardDataReader sdr = cmd.ExecuteReader()) 
                { 
                    while (sdr.Read())
                        { 
                            var c1Field = sdr.GetString(0); 
                            var c2Field = sdr.GetFieldValue<int>(1); 
                            var c3Field = sdr.GetFieldValue<Int64>(2);
                        } 
                 } 
           } 
    } 


<span data-ttu-id="5ff11-117">En önemli fark, çok parça bağlantılarının yapıdır.</span><span class="sxs-lookup"><span data-stu-id="5ff11-117">A key difference is the construction of multi-shard connections.</span></span> <span data-ttu-id="5ff11-118">Burada **SqlConnection** tek bir veritabanı üzerinde çalışır **MultiShardConnection** geçen bir ***parça koleksiyonunu*** giriş olarak.</span><span class="sxs-lookup"><span data-stu-id="5ff11-118">Where **SqlConnection** operates on a single database, the **MultiShardConnection** takes a ***collection of shards*** as its input.</span></span> <span data-ttu-id="5ff11-119">Parça parça eşlemesinden koleksiyonunu doldurun.</span><span class="sxs-lookup"><span data-stu-id="5ff11-119">Populate the collection of shards from a shard map.</span></span> <span data-ttu-id="5ff11-120">Sorgu kullanarak parça koleksiyonunu yürütülür **UNION ALL** semantiği tek bir genel sonuç birleştirin.</span><span class="sxs-lookup"><span data-stu-id="5ff11-120">The query is then executed on the collection of shards using **UNION ALL** semantics to assemble a single overall result.</span></span> <span data-ttu-id="5ff11-121">İsteğe bağlı olarak, satırın kaynaklandığı parça adını kullanarak çıktı eklenebilir **ExecutionOptions** özelliği komutu.</span><span class="sxs-lookup"><span data-stu-id="5ff11-121">Optionally, the name of the shard where the row originates from can be added to the output using the **ExecutionOptions** property on command.</span></span>
